# TurnLightDetection

TurnLightDetection is a YOLO-based CCTV turn-signal detection project. It trains an object detector to classify vehicle turn-signal states from CCTV frames and videos.

The goal is to help review CCTV footage of turning vehicles and identify whether the visible left or right turn signal is on or off. The project keeps the code, training utilities, and documentation in GitHub while storing large datasets and model artifacts outside the repository.

## Classes

The detector uses four classes:

| ID | Class |
|---:|---|
| 0 | `left_on` |
| 1 | `left_off` |
| 2 | `right_on` |
| 3 | `right_off` |

## Repository Contents

```text
.
├── audit_dataset.py           # Check YOLO labels, class counts, and train/val clip leakage
├── prepare_clip_split.py      # Rebuild a YOLO dataset split by clip id
├── train_day.py               # Train a YOLO model
├── validate_model.py          # Validate a trained model
├── predict_video.py           # Run image/video prediction or ByteTrack tracking
├── check_cuda.py              # Check PyTorch CUDA availability
├── install_cuda_pytorch.ps1   # Install PyTorch CUDA wheels and project dependencies
├── train_fast.ps1             # Fast baseline training command
├── train_balanced_3080.ps1    # RTX 3080-oriented training command
├── requirements.txt
└── cctv-turn-signal-dataset   # Dataset submodule
```

Large local artifacts are intentionally excluded from this repository, including raw clips, generated YOLO datasets, training runs, caches, and model checkpoints.

## Setup

Python 3.11 is recommended because PyTorch CUDA wheels may not be available for newer Python versions.

```powershell
conda create -n turn_signal python=3.11 -y
conda activate turn_signal
```

Install dependencies:

```powershell
pip install -r requirements.txt
```

For a CUDA-enabled PyTorch setup on Windows, run:

```powershell
.\install_cuda_pytorch.ps1
```

Check whether CUDA is available:

```powershell
python check_cuda.py
```

## Dataset

The dataset is managed separately from the code. Public code used to collect candidate CCTV clips is kept in the `cctv-turn-signal-dataset/` submodule. The collected and labeled data are shared through Google Drive.

Dataset sources:

| Data | Access |
|---|---|
| Data generated from `cctv-turn-signal-dataset/` | <https://drive.google.com/drive/folders/1NKHfG0ha_oyxFv2m64oFI7KpzLd6fbWR?usp=sharing> |
| Labeled dataset | <https://drive.google.com/drive/folders/1r7A_RUu204unDsdDivVN-3MLRTnw_ozb?usp=sharing> |

If cloning this repository from scratch, include submodules:

```powershell
git clone --recurse-submodules https://github.com/lindanceleaf/TurnLightDetection.git
```

If the repository was already cloned without submodules:

```powershell
git submodule update --init --recursive
```

This project expects YOLO-format data with a `data.yaml` file. Local dataset folders such as `raw_clips/`, `yolo_dataset/`, `yolo_dataset_day/`, and `yolo_dataset_day_clip_split/` are ignored by Git.

After downloading the labeled dataset, place it so that the default data file is available at:

```text
yolo_dataset_day/yolo_dataset_day/data.yaml
```

Alternatively, pass the dataset path explicitly with `--data`.

Audit an existing YOLO dataset:

```powershell
python audit_dataset.py --data yolo_dataset_day/yolo_dataset_day/data.yaml
```

Create a clip-level train/validation split to reduce leakage between train and validation frames:

```powershell
python prepare_clip_split.py --data yolo_dataset_day/yolo_dataset_day/data.yaml --out yolo_dataset_day_clip_split
```

Use `--overwrite` if the output folder already exists:

```powershell
python prepare_clip_split.py --data yolo_dataset_day/yolo_dataset_day/data.yaml --out yolo_dataset_day_clip_split --overwrite
```

## Training

Fast baseline:

```powershell
.\train_fast.ps1
```

RTX 3080-oriented balanced training:

```powershell
.\train_balanced_3080.ps1
```

Manual training example:

```powershell
python train_day.py --data yolo_dataset_day_clip_split/data.yaml --model yolov8s.pt --imgsz 960 --batch 16 --epochs 50 --device 0 --name yolov8s_img960_b16_balanced
```

Training outputs are written under:

```text
runs/turn_signal_day/<run_name>/
```

The best checkpoint is usually:

```text
runs/turn_signal_day/<run_name>/weights/best.pt
```

## Validation

```powershell
python validate_model.py --model runs/turn_signal_day/yolov8s_img960_b16_balanced/weights/best.pt --data yolo_dataset_day_clip_split/data.yaml --imgsz 960 --batch 8 --device 0
```

Validation plots and metrics are generated by Ultralytics under the run output directory.

## Prediction and Tracking

Run prediction on an image, folder, or video:

```powershell
python predict_video.py --model runs/turn_signal_day/yolov8s_img960_b16_balanced/weights/best.pt --source path/to/input
```

Run video tracking with ByteTrack:

```powershell
python predict_video.py --model runs/turn_signal_day/yolov8s_img960_b16_balanced/weights/best.pt --source path/to/video.mp4 --track
```

Prediction outputs are written under:

```text
runs/predict_turn_signal/
```

## Notes

- Model checkpoints (`*.pt`) are ignored and should be shared separately if needed.
- Training outputs under `runs/` are ignored.
- Dataset folders are ignored to keep the GitHub repository lightweight.
- The recommended workflow is to keep code in this repository and keep datasets/checkpoints in a separate storage location or dataset repository.
